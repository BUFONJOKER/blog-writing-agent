2.1 — Initialize FastMCP project

Est. Hours2
PhasePhase 2 - FastMCP
PriorityHigh
SprintSprint 1
StatusNot Started

Overview

Set up the FastMCP server project from scratch. FastMCP is a Python framework that makes it trivial to build MCP-compliant tool servers. The server will host all research and utility tools that LangGraph will call during agent execution.

Why This Matters

The MCP server is an independently deployable service. Keeping it separate from the backend means you can update, restart, or swap tools without touching the agent. It also means multiple agents can reuse the same tools.

Steps

  1. Navigate to the mcp_server/ folder
  1. Create a Python virtual environment: python -m venv .venv && source .venv/bin/activate
  1. Install dependencies: pip install fastmcp httpx beautifulsoup4 tavily-python python-dotenv
  1. Create requirements.txt by running pip freeze > requirements.txt
  1. Create server.py with the following structure:
    • Import FastMCP from fastmcp
    • Create the app: mcp = FastMCP('blog-research-tools')
    • Add a if __name__ == '__main__': mcp.run() block
  1. Create a tools/ subfolder with __init__.py where individual tool functions will live
  1. Create a config.py that loads all env vars using python-dotenv
  1. Test the server starts: fastmcp run server.py — you should see a startup message with no errors

Acceptance Criteria

  • fastmcp run server.py starts without errors
  • Server is accessible at http://localhost:8000 (default FastMCP port)
  • requirements.txt exists with all pinned versions
  • Project structure is clean with tools/ subfolder ready

Notes

FastMCP defaults to port 8000. Change to port 8001 to avoid conflict with the FastAPI backend. Use mcp.run(port=8001) or set via env var.
