5.9 — Add error handling and loading states

Est. Hours3
PhasePhase 5 - Frontend
PriorityMedium
SprintSprint 5
StatusNot Started

Overview

Add comprehensive error handling, loading skeletons, and toast notifications across the frontend to create a polished, production-quality user experience.

Steps

  1. Install a toast library: npm install sonner
  1. Add <Toaster /> to the root layout in app/layout.tsx
  1. Add toast notifications for:
    • Plan approved: 'Plan approved! Starting to write...'
    • Draft ready: 'Your draft is ready for review!'
    • Generation complete: 'Blog post complete!'
    • Any API error: show the error message in a red toast
  1. Add loading skeletons for the history page:
    • When runs are loading, show 3-4 placeholder cards with animated gray shimmer
  1. Add error boundary component:
    // Create src/components/ErrorBoundary.tsx
// Wrap major page sections to catch React rendering errors
// Show a friendly 'Something went wrong' with a retry button
  1. Handle SSE reconnection: if the EventSource emits an error event, show a banner: 'Connection lost. Reconnecting...' with a retry button
  1. Validate form inputs on the prompt page:
    • Prompt cannot be empty
    • Prompt must be at least 10 characters
    • Show validation error inline, not just on submit

Acceptance Criteria

  • Toast appears for all major state transitions
  • Loading skeletons display while data is being fetched
  • SSE disconnection is handled gracefully with a visible reconnect option
  • Form validation prevents empty submissions
  • Error boundary catches crashes and shows a recovery UI
