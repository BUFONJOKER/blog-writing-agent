2.9 — Dockerize the MCP server

Est. Hours2
PhasePhase 2 - FastMCP
PriorityMedium
SprintSprint 6
StatusNot Started

Overview

Create a Dockerfile for the FastMCP server so it can be built and run as a container. This enables consistent deployment and makes the docker-compose setup in Phase 6 straightforward.

Steps

  1. Create mcp_server/Dockerfile:
    FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 8001
HEALTHCHECK --interval=30s --timeout=10s CMD curl -f http://localhost:8001/health || exit 1
CMD ["python", "server.py"]
  1. Add a /health endpoint to server.py that returns {'status': 'ok'} — use FastMCP's underlying FastAPI app: mcp.app.get('/health')(lambda: {'status': 'ok'})
  1. Add mcp_server/ to .dockerignore: exclude .venv, __pycache__, tests/, .env
  1. Build the image: docker build -t blog-agent-mcp .
  1. Run locally: docker run -p 8001:8001 --env-file .env blog-agent-mcp
  1. Test the health check: curl http://localhost:8001/health — should return {'status': 'ok'}
  1. Test a tool call against the running container

Acceptance Criteria

  • docker build completes without errors
  • Container starts and /health returns 200
  • Tool calls work correctly against the containerized server
  • Image size is under 500MB

Notes

Use python:3.11-slim (not full) to keep the image small. The slim image is sufficient since we only need Python and pip.
