3.5 — Build the Research Executor node

Est. Hours4
PhasePhase 3 - LangGraph
PriorityHigh
SprintSprint 3
StatusNot Started

Overview

Build the most complex node in the system: the Research Executor. This is a ReAct-style agent that loops through search and fetch tool calls, accumulating research until it has enough information.

Why This Matters

This node is where the actual web research happens. It must be smart enough to follow interesting results with deeper fetches, while also knowing when to stop.

Steps

  1. Open agent/nodes/researcher.py
  1. Import the MCP tools from agent/tools.py
  1. Create a research-focused LLM: ChatOpenAI(model='gpt-4o', temperature=0)
  1. Bind the web_search and fetch_page tools to the LLM: llm_with_tools = llm.bind_tools([web_search_tool, fetch_page_tool])
  1. Build the researcher system prompt:
    You are a research assistant. Your job is to gather comprehensive information for a blog post.
You have access to: web_search (search the web) and fetch_page (read a full webpage).

Strategy:
1. Use web_search for each query in the provided list
2. For the 2-3 most relevant results, use fetch_page to get full content
3. Stop when you have enough information to write a comprehensive blog post
4. Maximum 8 tool calls total

Research queries to execute: {state['research_queries']}
  1. Implement the ReAct loop using LangGraph's ToolNode:
    • Add the researcher LLM as one node
    • Add ToolNode([web_search_tool, fetch_page_tool]) as another node
    • Add a conditional edge: if last message has tool calls → go to ToolNode, else → go to summarizer
  1. Cap tool calls at 8 using a counter in state
  1. Collect all tool results and store them in state['research_results']

Acceptance Criteria

  • Executes at least one web_search per research query
  • Fetches full content for top results using fetch_page
  • Stops at 8 tool calls maximum
  • Stores all results in state['research_results']
  • Does not crash on tool errors (fetch failures, etc.)

Notes

This is where most debugging will happen. Use LangSmith to trace every tool call and see what data is being collected.
