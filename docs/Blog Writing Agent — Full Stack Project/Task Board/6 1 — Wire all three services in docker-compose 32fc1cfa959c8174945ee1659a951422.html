6.1 — Wire all three services in docker-compose

Est. Hours4
PhasePhase 6 - DevOps
PriorityHigh
SprintSprint 6
StatusNot Started

Overview

Create the root docker-compose.yml that starts all five services (frontend, backend, MCP server, PostgreSQL, Redis) with a single command. This is the primary way to run the full stack locally.

Steps

  1. Open docker-compose.yml at the project root
  1. Define all services:
    version: '3.9'

services:
  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: blogagent
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    ports: ['5432:5432']
    volumes: ['pgdata:/var/lib/postgresql/data']
    healthcheck:
      test: ['CMD', 'pg_isready', '-U', 'postgres']
      interval: 10s
      timeout: 5s
      retries: 5
  
  mcp_server:
    build: ./mcp_server
    ports: ['8001:8001']
    env_file: ./mcp_server/.env
    healthcheck:
      test: ['CMD', 'curl', '-f', 'http://localhost:8001/health']
      interval: 30s
  
  backend:
    build: ./backend
    ports: ['8000:8000']
    env_file: ./backend/.env
    environment:
      DATABASE_URL: postgresql+asyncpg://postgres:postgres@postgres:5432/blogagent
      MCP_SERVER_URL: http://mcp_server:8001
    depends_on:
      postgres:
        condition: service_healthy
      mcp_server:
        condition: service_healthy
  
  frontend:
    build: ./frontend
    ports: ['3000:3000']
    environment:
      NEXT_PUBLIC_BACKEND_URL: http://localhost:8000
    depends_on:
      - backend

volumes:
  pgdata:
  1. Add Dockerfiles for backend and frontend (similar pattern to MCP server Dockerfile from Task 2.9)
  1. For frontend Dockerfile: use multi-stage build with node:20-alpine as base, build with npm run build, serve with npm start
  1. Test: docker-compose up --build — all services should start healthy
  1. Run the end-to-end test script against the compose stack

Acceptance Criteria

  • docker-compose up starts all services without errors
  • Services start in correct dependency order (postgres → backend, mcp_server → backend → frontend)
  • Health checks pass for all services
  • Frontend at localhost:3000 can successfully call backend at localhost:8000
  • Database persists between container restarts using the named volume
