Est. Hours4

Overview

Write an automated end-to-end test script that runs the full blog generation flow programmatically, auto-approving all HIL checkpoints, and verifies the final output is a valid blog post.

Why This Matters

Manual testing every time you change something is slow and error-prone. This script catches integration bugs in CI/CD before they reach production.

Steps

  1. Create tests/e2e_test.py
  1. Write the test flow:
    import httpx
import asyncio
import time

BASE_URL = 'http://localhost:8000'

async def run_e2e_test():
    # 1. Start a blog run
    resp = await client.post('/blog/start', json={
        'prompt': 'A beginner guide to Python type hints',
        'user_id': 'e2e-test-user'
    })
    thread_id = resp.json()['thread_id']
    print(f'Started run: {thread_id}')
    
    # 2. Poll until HIL Checkpoint 1
    for _ in range(30):  # max 5 minutes
        await asyncio.sleep(10)
        status = await client.get(f'/blog/{thread_id}/status')
        data = status.json()
        print(f'Status: {data["status"]} | Node: {data.get("current_node")}')
        
        if data['awaiting_human'] and data['interrupt_type'] == 'plan_review':
            # Auto-approve the plan
            await client.post(f'/blog/{thread_id}/review', json={'action': 'approve'})
            print('Auto-approved plan')
            break
    
    # 3. Poll until HIL Checkpoint 2
    for _ in range(30):
        await asyncio.sleep(10)
        status = await client.get(f'/blog/{thread_id}/status')
        data = status.json()
        
        if data['awaiting_human'] and data['interrupt_type'] == 'final_review':
            await client.post(f'/blog/{thread_id}/review', json={'action': 'approve'})
            print('Auto-approved draft')
            break
    
    # 4. Poll until complete
    for _ in range(10):
        await asyncio.sleep(5)
        status = await client.get(f'/blog/{thread_id}/status')
        if status.json()['status'] == 'complete':
            break
    
    # 5. Fetch and verify result
    result = await client.get(f'/blog/{thread_id}/result')
    post = result.json()['final_post']
    
    assert len(post) > 500, 'Blog post is too short'
    assert '# ' in post, 'Blog post has no title'
    assert result.json()['word_count'] > 300, 'Word count too low'
    print(f'SUCCESS: {result.json()["word_count"]} words generated')
  1. Run against local compose stack: python tests/e2e_test.py
  1. Add to a GitHub Actions workflow that runs on every push to main

Acceptance Criteria

  • Test completes without errors against the compose stack
  • Final blog post has at least 500 characters and a markdown title
  • Test runs in under 5 minutes
  • Test fails loudly with a clear message if any step takes too long
