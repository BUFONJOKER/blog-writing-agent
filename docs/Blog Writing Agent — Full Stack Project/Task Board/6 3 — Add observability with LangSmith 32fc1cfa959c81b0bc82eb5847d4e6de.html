6.3 — Add observability with LangSmith

Est. Hours2
PhasePhase 6 - DevOps
PriorityHigh
SprintSprint 6
StatusNot Started

Overview

Connect LangSmith tracing to the LangGraph backend so every LLM call, tool call, and node transition is traced and inspectable in the LangSmith UI.

Why This Matters

Without observability, debugging agent failures is like debugging a black box. LangSmith shows you exactly what the LLM received as input, what it returned, how many tokens it used, and how long each step took.

Steps

  1. Sign up at smith.langchain.com and create an API key
  1. Add to backend/.env:
    LANGCHAIN_TRACING_V2=true
LANGCHAIN_API_KEY=your_key_here
LANGCHAIN_PROJECT=blog-agent
  1. LangGraph automatically picks up these env vars — no code changes needed
  1. Run a complete blog generation and then visit the LangSmith UI
  1. Find your trace and verify you can see:
    • Each node in the graph as a span
    • The LLM input and output for each node
    • Tool calls with their inputs and outputs
    • Token counts per call
    • Total latency and cost estimate
  1. Set up a LangSmith dataset for evaluation: add 5 test prompts with expected output characteristics
  1. Add metadata to traces:
    config = {
  'configurable': {'thread_id': thread_id},
  'metadata': {'user_id': user_id, 'prompt_preview': prompt[:50]}
}

Acceptance Criteria

  • LangSmith dashboard shows a trace for every blog run
  • Each trace shows all nodes, LLM calls, and tool calls
  • Token counts are visible per node
  • Traces can be filtered by project name
  • No sensitive data (API keys, full user data) appears in traces
