5.2 — Build the prompt input page

Est. Hours3
PhasePhase 5 - Frontend
PriorityHigh
SprintSprint 5
StatusNot Started

Overview

Build the homepage where users enter their blog topic and optional configuration (tone, audience), then submit to start the agent. This is the first thing users see.

Steps

  1. Open src/app/page.tsx
  1. Build a centered layout with:
    • A large heading: 'What do you want to write about?'
    • A <textarea> with placeholder 'Enter your blog topic or idea...', styled generously (min 120px height)
    • An optional settings section (collapsible or always visible):
      • Tone dropdown: Informative, Casual, Technical, Persuasive
      • Target audience text input
    • A 'Start Writing' button that is disabled while the request is loading
  1. On submit:
    const handleSubmit = async () => {
  setLoading(true)
  try {
    const { data } = await api.post('/blog/start', {
      prompt,
      user_id: 'test-user',  // Replace with real auth in Sprint 6
      tone,
      target_audience: audience
    })
    router.push(`/blog/${data.thread_id}`)
  } catch (error) {
    setError('Failed to start. Please try again.')
  } finally {
    setLoading(false)
  }
}
  1. Add a character count under the textarea (max 500 chars recommended)
  1. Add example prompts as clickable chips: 'The future of remote work', 'Beginner guide to TypeScript', 'How to start investing'
  1. Style with Tailwind — clean, minimal, centered on the page

Acceptance Criteria

  • Form submits correctly and redirects to /blog/{threadId}
  • Button is disabled during loading with a spinner
  • Error message shows on API failure
  • Example prompt chips populate the textarea on click
  • Mobile responsive layout
