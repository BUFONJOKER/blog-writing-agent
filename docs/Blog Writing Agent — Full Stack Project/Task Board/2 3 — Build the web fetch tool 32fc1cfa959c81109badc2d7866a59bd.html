2.3 — Build the web fetch tool

Est. Hours3
PhasePhase 2 - FastMCP
PriorityHigh
SprintSprint 1
StatusNot Started

Overview

Create a tool that fetches the full content of a web page and returns clean, readable text. The research node calls this after web_search to get the complete content of the most relevant results.

Why This Matters

Search result snippets are often too short to be useful. Fetching the full page gives the research node enough detail to extract specific facts, statistics, and quotes for the blog post.

Steps

  1. Create tools/fetch_page.py
  1. Define the tool with @mcp.tool() decorator:
    • Function name: fetch_page
    • Parameters: url: str
    • Return type: dict
  1. Use httpx.AsyncClient (async for better performance) with a 10-second timeout
  1. Set a realistic User-Agent header to avoid bot detection: 'Mozilla/5.0 (compatible; BlogAgent/1.0)'
  1. On successful fetch, pass HTML to BeautifulSoup(html, 'html.parser')
  1. Extract title: soup.find('title').text
  1. Remove noise: soup.find_all(['script', 'style', 'nav', 'footer', 'header', 'aside']) — decompose all of these
  1. Extract body text: soup.get_text(separator='\n', strip=True)
  1. Truncate to 4000 characters to stay within LLM context limits
  1. Return: {'url': url, 'title': title, 'content': cleaned_text, 'char_count': len(cleaned_text)}
  1. Handle errors: timeout → return error dict; non-200 status → return error dict with status code; invalid URL → return error dict
  1. Register in server.py

Acceptance Criteria

  • Returns clean text from any public URL within 10 seconds
  • Content is stripped of navigation, scripts, and other noise
  • Never crashes on bad URLs or timeouts — always returns a dict
  • Truncation keeps content within usable LLM context size

Notes

Some sites block scrapers. If you get frequent 403 errors, consider using Tavily's get_search_context which already handles content extraction. For paywalled sites, the tool will return minimal content — this is acceptable behavior.
